## Configuration

**Environment:**
- `.env.local` - Local development secrets (git-ignored)
  - `CONVEX_DEPLOYMENT` - Convex project deployment identifier
  - `VITE_CONVEX_URL` - Backend API endpoint
  - `VITE_CONVEX_SITE_URL` - Convex site URL for OAuth redirects
  - `AUTH_RESEND_KEY` - Resend API key for email sending
  - `VAPID_SUBJECT`, `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY` - Web Push credentials
  - `SITE_URL` - Deployed site URL for invite links

**Build:**
- `vite.config.ts` - Vite configuration with React plugin, path aliases (`@/*`, `@shared/*`), HTTPS for dev
- `tailwind.config.js` - Theme colors, animations, dark mode (selector + class strategy)
- `postcss.config.js` - PostCSS with Tailwind and Autoprefixer
- `tsconfig.json` - Root TypeScript config with path aliases
- `tsconfig.app.json` - Frontend-specific TypeScript settings
- `tsconfig.node.json` - Build tool TypeScript settings
- `convex/tsconfig.json` - Convex backend TypeScript (strict mode, ESNext target)
- `.eslintrc.cjs` - ESLint rules for TypeScript and React
- `.prettierrc` - Prettier formatter (100 char print width)

**Convex:**
- `convex.config.ts` - Convex modules: ProseMirror Sync, Presence
- `convex/auth.config.ts` - Auth provider domain configuration
- `convex/_generated/` - Auto-generated types and API (do not edit)

## Platform Requirements

**Development:**
- Node.js 20.x+ (ES modules support, modern npm)
- HTTPS localhost certs (generated by mkcert 3.2.0)
- Environment variables for Convex, Resend, Web Push

**Production:**
- Deployment target: Convex cloud backend + static hosting (Vercel, Netlify, or similar)
- Environment variables deployed to hosting platform
- VAPID keys for push notifications required
